### 3. 뉴스레터 형태로 헤드라인, 내용 정리해서 매일 아침 관련자들 자동 송부



## 1. 기술 스택 및 아키텍처 탐색용 프롬프트
이 프롬프트는 시스템을 구축하기 위해 어떤 오픈소스와 논문 기법을 써야 할지 물어볼 때 좋습니다.

Prompt:
"I want to build an automated RAG-based system for managing official project letters (EPC contracts). The goal is to:

Automatically extract, translate, and summarize incoming PDF letters.

Detect specific Action Items and Due Dates requested by the sender.

Generate a daily 'News-style' summary for stakeholders.

Please find recent research papers or technical blogs (2024-2026) regarding 'Automated Action Item Extraction from Legal Documents', 'Multi-modal RAG for PDF Table/Highlighter detection', and 'LLM-based Email Newsletter Automation'. Focus on solutions that can be deployed locally using high-end GPUs like RTX 5090."

## 2. 구체적인 기능 구현 방법론(하이라이트/요약) 탐색용
문라이트(Moonlight)처럼 중요한 곳에 하이라이트를 치는 기능은 'Visual RAG'나 'Layout-aware LLM' 기술이 필요합니다. 이를 찾기 위한 프롬프트입니다.

Prompt:
"Search for practical implementation methods for 'Document Visual Highlighting' using LLMs. I need to replicate features similar to 'Moonlight' where the AI highlights critical parts of an official letter.
Specifically, look for papers or GitHub projects on:

Layout-aware PDF parsing (e.g., Docling, Unstructured, or Marker).

Techniques to map LLM's summarized text back to original PDF coordinates (Bounding Boxes).

Best practices for Zero-shot Translation of technical engineering terms in FLNG/Shipbuilding contexts."

## 3. 뉴스레터 및 워크플로우 자동화 탐색용
매일 아침 자동으로 메일을 보내는 파이프라인(Agentic Workflow)을 찾기 위한 프롬프트입니다.

Prompt:
"Find the best workflow patterns for an 'AI News-letter Agent'. The system should:

Scan a specific directory or Vector DB for new letters received within the last 24 hours.

Format a structured Markdown/HTML summary including: Headline, Core Content, Action Items, and Urgency.

Integrate with Python-based email automation (SMTP) or Slack/Teams.
Search for recent articles on 'Agentic RAG Workflows' and 'Automated Project Reporting with LLMs'."

## 💡 팁: Perplexity 활용 노하우
Pro 모드 활용: Perplexity의 Pro 모드를 켜고 질문하면, 검색 결과를 종합하여 더 논리적인 아키텍처를 제안해 줍니다.

Focus 기능: 검색 범위를 **'Academic'**으로 설정하면 논문 위주로, **'Writing'**이나 **'All'**로 하면 실전 코드 위주로 찾아줍니다.

파일 업로드: 현재 회사에서 사용하는 공문 양식(비민감 정보만 포함된 샘플)을 Perplexity에 올리고 **"이 문서에서 Action Item과 날짜를 추출하기 위한 최적의 프롬프트를 짜줘"**라고 요청해 보세요.
