## 5. 프롬프트 엔지니어링 핵심

### 프롬프트 3요소

| 요소 | 영문 | 설명 |
|------|------|------|
| 목적 | Task | LLM이 수행해야 할 핵심 과업 |
| 문맥 | Context | 과업이 필요한 목적과 배경 정보 |
| 지시사항 | Instructions | 결과물의 형식 / 조건 / 세부 지시 |

> Task만으로도 답변 생성은 가능하나, **Context + Instructions**를 추가하면 정확성과 만족도가 높아짐.

### Context(맥락/배경)의 역할

- 지시한 과업이 **왜 필요한지**, **어떤 상황인지** 설명
- LLM이 목적과 배경을 이해 → 더 정교하고 실행 가능한 결과 도출
- 단순 아이디어가 아닌, **실제 문제를 반영한 해결책** 가능성 ↑

### 예시 프롬프트
```
현재 노인 인구가 증가하고 은퇴 시기도 늦춰지는 상황이야.
정년 연장, 노인 취업, 복지 예산 등 새로운 정책이 필요한 분위기야.
이와 관련하여 노인 정책 입안을 위한 아이디어를 제시해줘.
```

### Few-Shot Prompting

- ✅ 적절한 예시는 AI가 의도를 더 정확하게 파악하도록 도움
- ⚠️ 예시만 단독 제시 시 AI가 어느 패턴(의미 vs 구조)에 초점을 맞출지 혼란스러워 함
- ⭐ **Best Practice** : 충분한 설명 + 예시를 함께 제시

## 6. Streamlit

> `app.py`에 모든 기능을 넣고 서버처럼 구동. 인터넷 주소만 있으면 누구나 접속 가능.

### 설치
```bash
pip install streamlit
# 또는
python -m pip install streamlit
```

### 실행 순서

1. VS Code에서 `app.py` 생성
2. 모든 코드를 `app.py`에 작성
3. **터미널**에서 실행 (코드창에서 실행 시 안 멈춤)
```bash
streamlit run app.py
# 또는
streamlit run .\app.py
```

### %%writefile로 app.py 덮어쓰기

셀 최상단에 `%%writefile app.py`를 붙이면 실행 시 파일이 자동으로 덮어써짐.
```python
%%writefile app.py
import streamlit as st

st.title("Welcome to Streamlit!")
st.header("Exploring the basics")
st.write("The most important libraries in Python")
st.write('- NumPy')
st.write('- Pandas')
st.write('- Matplotlib')
```

이미 app.py를 만들어둔 상태라 여기 덮어 씌우는 %%writefile app.py 이용해서 해줄래


### Text Elements 요약

| 함수 | 설명 |
|------|------|
| `st.markdown` | Markdown 표시. 색상, 이모지, LaTeX 수식 지원 |
| `st.title` | 앱 제목. 문서당 하나 권장 |
| `st.header` | 헤더 텍스트. `divider` 옵션으로 색상 구분선 추가 가능 |
| `st.subheader` | 서브헤더. `divider` 옵션 지원 |
| `st.badge` | 아이콘 + 레이블이 있는 색상 배지 |
| `st.caption` | 소형 텍스트. 캡션 · 각주 · 부연 설명용 |
| `st.echo` | `with` 블록 안의 코드를 출력하고 동시에 실행. 튜토리얼에 유용 |
| `st.latex` | LaTeX 형식 수학 수식 표시 |
| `st.divider` | 수평 구분선. `st.write("---")`와 동일 |
| `st.html` | iframe 없이 HTML 삽입. JS는 기본 무시 |
| `st.components.v1.html` | iframe 안에 HTML 렌더링. JS 포함 HTML 실행 시 사용 |

### 웹스크래핑 소개
◼︎ 웹스크래핑이란?
✓웹사이트에 있는 데이터를 자동으로 수집하는 기술
✓웹페이지 -> HTML 분석 -> 원하는 데이터 추출 -> 저장
◼︎ 웹스크래핑 활용 예시
✓가격 비교 시스템 (쿠팡, 네이버 쇼핑)
✓뉴스 모니터링 시스템
✓주식/코인 데이터 수집
✓리뷰 분석 (감성 분석과 연결 가능)
◼︎ 웹스크래핑 동작 원리
✓웹페이지 요청(Request)
✓서버에 요청해서 HTML 가져오기
✓파이썬에서는 주로 requests를 많이 사용함
✓HTML 구조 분석 (Parsing)
✓웹페이지는 HTML 구조로 되어 있음
✓태그(<div>, <a>, <span> 등) 기반으로 분석
✓데이터 추출
✓원하는 요소 선택 (CSS selector, XPath 등)
✓예: 상품명, 가격, 날짜
✓데이터 저장
✓CSV, DB, Excel 등으로 저장
◼︎ 기술보다 법과 윤리 준수 중요
✓robots.txt (수집 허용 여부)
✓크롤러 사용자에게 “해도 되는지” 알려주는 규칙서
✓서비스 이용약관
✓과도한 요청 금지 (서버 부하)
✓개인정보 수집 금지
