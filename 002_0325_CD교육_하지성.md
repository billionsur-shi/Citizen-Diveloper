## 5. 🧠 프롬프트 엔지니어링 핵심

### 프롬프트 3요소
| 요소 | 설명 |
|------|------|
| **목적 (Task)** | LLM이 수행해야 할 핵심 과업 |
| **문맥 (Context)** | 과업이 필요한 목적과 배경 정보 |
| **지시사항 (Instructions)** | 결과물의 형식 / 조건 / 세부 지시 |

> Task만으로도 답변 생성은 가능하나, Context + Instructions를 추가하면 **정확성과 만족도가 높아짐**

### Context(맥락/배경)의 역할
- 지시한 과업이 **왜 필요한지**, **어떤 상황인지** 설명
- LLM이 목적과 배경을 이해 → 더 정교하고 실행 가능한 결과 도출
- 단순 아이디어가 아닌, **실제 문제를 반영한 해결책** 가능성 ↑

### 예시 프롬프트
```
현재 노인 인구가 증가하고 은퇴 시기도 늦춰지는 상황이야.
정년 연장, 노인 취업, 복지 예산 등 새로운 정책이 필요한 분위기야.
이와 관련하여 노인 정책 입안을 위한 아이디어를 제시해줘.
```

### Streamlit 소개
- 서버를 돌리는거라고 생각하면 됨
- app.py에 모든 기능을 넣고 돌릴거임
- streamlit app.py 로 구동 (계속돌림)
- 인터넷 주소만 있으면 누구나 접속해서 사용
- 삼성중공업 정책때문에 인터넷에 올리진 못함

### streamlit 순서
- vscode에서 app.py 생성
- 여기에 모든 코드 넣는다

### Streamlit 설치

### 하기 명령어를 통해 app.py 덮어 씌움 (여러 풀을 자유자재로 이동 가능)
- %%writefile app.py 기능 때문임
- 
%%writefile app.py
import streamlit as st
st.title("Welcome to Streamlit!")
st.header("Exploring the basics")
st.write("The most important libraries at Python")
st.write('- Numpy')
st.write('- Pandas')
st.write('- Matplotlib')

### Text Elements
- st.markdown:
    - Markdown 형식의 문자열 표시
    - 색상, 이모지, LaTeX 수식 지원
- st.title:
    - 앱의 제목 표시
    - 문서당 하나의 st.title() 사용 권장
- st.header:
    - 헤더 형식의 텍스트 표시
    - divider 옵션으로 색상 구분선을 추가할 수 있음
- st.subheader:
    - 서브헤더 형식의 텍스트 표시
    - divider 옵션 지원
- st.badge:
    - 아이콘과 레이블이 있는 색상 배지 표시
- st.caption:
    - 작은 글꼴로 텍스트 표시
    - 캡션, 각주, 부연 설명 등에 사용
- st.echo:
    - with 블록 안의 코드를 화면에 출력하고 동시에 실행
    - 튜토리얼 등에 유용
- st.latex:
    - LaTeX 형식으로 수학 수식 표시
- st.divider:
    - 수평 구분선 표시
    - st.write("---")와 동일한 효과
- st.html:
    - iframe 없이 HTML 삽입
    - 앱 내에 직접 HTML을 삽입하고, 기본적으로 JavaScript 무시
- st.components.v1.html:
    - iframe 안에 HTML 렌더링
    - JavaScript 포함 HTML을 iframe 안에서 실행할 때 사용
